# Neutron Star EoS ML Pipeline

This repository contains the Master Pipeline Orchestrator for Neutron Star Equation of State (EoS) Inference. It pairs a parallelized Tolman-Oppenheimer-Volkoff (TOV) solver with Random Forest implementations to map macroscopic neutron star observables to their microscopic physical phases.

## Features
- **High-Performance Physics Engine**: Generates up to 10,000 dense EoS solutions matching physical constraints (e.g., causality, conformal limits, thermodynamic stability).
- **Parquet Data Infrastructure**: Uses Apache Parquet via `pyarrow` for extremely fast serialization, storage, and retrieval of massive, high-resolution phase space topologies.
- **Machine Learning Analysis**: Implements robust ensemble classifiers (LightGBM/XGBoost logic/MLP) paired with Shapley additive explanations (SHAP) to interpret observational dependencies.

## Setup
It is recommended to use a virtual environment:

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate
# Unix/MacOS
source .venv/bin/activate

pip install -r requirements.txt
```

## Usage

### 1. The Main Inference Pipeline
Run the master pipeline to execute the parallel physics generation, evaluate the machine learning model, and produce all final insights.

```bash
python main.py
```
Outputs: 
- `data/physics_test_dataset.parquet`: The generated physics samples.
- `models/`: Saved `joblib` model objects.
- `plots/`: Comprehensive publication-ready visualizations of the topological phase spaces.

### 2. Isolated Physics Pipeline
To rigorously test the Equation of State construction independently of the ML pipeline:

```bash
python physics_main.py
```

## Directory Structure
- `src/physics/`: Physics solvers and parallel wrappers to build the EoS and integrate the TOV equations.
- `src/ml_pipeline/`: Machine learning workflows including model training and evaluation interfaces.
- `src/visualize/`: Extensive Matplotlib plotting tools customized via `style_config`.
- `data/`: Autogenerated dataset outputs (excluded from version control).
- `models/`: ML checkpoints and state (excluded from version control).
- `plots/`: Directory for generated `.pdf` figures and analyses (excluded from version control).
